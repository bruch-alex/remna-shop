# Remnawave Shop (DDD Document)

## 1. Ubiquitous Language

Plan – A purchasable VPN product with a defined price, duration, bandwidth limit, device limit.
Subscription – A user's active entitlement to a Plan. Has a start date, expiry date, and a current status (ACTIVE, EXPIRING, EXPIRED, SUSPENDED).
VpnProfile – The internal mirror of a Remnawave user record. Stores the Remnawave user ID, generated access URL, and current traffic usage.
Access URL – The connection string delivered to the user to configure their VPN client. Generated by Remnawave upon provisioning.
Order – A user's intent to purchase a Plan. Transitions from PENDING to PAID to COMPLETED.

## 2. Bounded Contexts

The application is divided into six bounded contexts. 
Each context is a self-contained module with its own domain model, application services, and persistence layer. 
No context accesses another context's database tables directly.

- User – Identity, authentication, user profiles
- Catalog – VPN plan definitions, pricing, regions
- Order – Cart, checkout flow, order lifecycle
- Payment – Payment processing, Stripe webhooks
- Subscription – Subscription lifecycle, renewal scheduling
- VpnProfile – VPN provisioning via Remnawave API

## 3. Domain Model
### 3.1 User Context

Owns everything related to identity. The User is the root aggregate. 
Authentication is handled via JWT access tokens and refresh tokens stored in the database. 
The context exposes a UserService used by other modules to look up user information.

## 4. Package Structure

Each module follows the same internal layout:

```

    user/

            dto/ <- DTOs for user requests and responses
        domain/ <- Aggregates (models)
            User.java <- Entity
            RefreshToken.java <- Entity
            UserRepository.java <- interface for database access
        infrastructure/ <- database access

    order/
```
